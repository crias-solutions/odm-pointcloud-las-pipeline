# ODM Point Cloud LAS Pipeline - Usage Guide

A modular Python-orchestrated photogrammetry processing pipeline that converts drone RGB imagery to georeferenced LAS/LAZ point clouds using OpenDroneMap (ODM).

---

## Installation Options

This pipeline has **3 installation tiers** depending on your environment:

---

### Tier 1: Minimal (Python Only) ✅ Available Now

Required: Python 3.12+

```bash
pip install -r requirements.txt
```

**What works:**
- ✅ Preflight validation
- ✅ Geospatial validation  
- ✅ geo.txt generation
- ❌ Reconstruction (needs Docker)
- ❌ LAS/LAZ processing (needs PDAL)

---

### Tier 2: With Docker

Required: Python 3.12+ + Docker daemon running

```bash
# Install Docker (Ubuntu/Debian)
sudo apt-get update
sudo apt-get install -y docker.io
sudo service docker start

# Verify Docker is running
docker --version
docker run hello-world
```

**Note:** Docker requires a Linux kernel with container support. Some environments (containers, VMs, CI/CD) may not support Docker.

**What works:**
- ✅ All Tier 1 features
- ✅ ODM Reconstruction
- ✅ Reconstruction validation
- ❌ LAS/LAZ processing (needs PDAL)

---

### Tier 3: Full Pipeline

Required: Python 3.12+ + Docker + PDAL

```bash
# Install PDAL
# Ubuntu:
sudo apt-get install pdal

# macOS:
brew install pdal

# Verify
pdal --version
```

**What works:**
- ✅ All features
- ✅ Full pipeline end-to-end

---

## Quick Start (Python Only)

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Run validation
python -m src.cli.main validate tests/fixtures/mock_dataset --stage preflight

# 3. Generate geo.txt
python -m src.cli.main geotxt tests/fixtures/mock_dataset
```

## Dataset Structure

Your dataset must follow this structure:

```
dataset_id/
├── raw_images/              # Drone images (JPEG, PNG, TIFF)
│   ├── image_001.jpg
│   ├── image_002.jpg
│   └── ...
├── flight_logs/
│   └── log.csv              # Flight log with GPS data
├── metadata/
│   └── dataset_config.yaml  # Optional dataset config
├── processing/
│   ├── geo.txt              # Generated by pipeline
│   └── odm/                 # ODM output
└── outputs/
    ├── pointcloud.las       # Normalized LAS
    ├── pointcloud.laz       # Compressed LAZ
    └── reconstruction_report.json
```

### Flight Log Format (log.csv)

```csv
timestamp,latitude,longitude,altitude
2024-01-15T10:00:00,40.7128,-74.0060,50.0
2024-01-15T10:00:02,40.7129,-74.0061,50.5
```

## CLI Commands

### Full Pipeline Run

Process a dataset through all stages:

```bash
python -m src.cli.main run /path/to/dataset_id --strategy odm
```

### Individual Commands

#### 1. Validate Dataset (Preflight)

Check dataset structure before processing:

```bash
python -m src.cli.main validate /path/to/dataset --stage preflight
```

#### 2. Generate geo.txt

Create geo.txt from flight log:

```bash
python -m src.cli.main geotxt /path/to/dataset
```

#### 3. Validate geo.txt

Check geospatial data:

```bash
python -m src.cli.main validate /path/to/dataset --stage geospatial
```

#### 4. Run Reconstruction

Execute ODM reconstruction:

```bash
python -m src.cli.main reconstruct /path/to/dataset --strategy odm
```

#### 5. Validate Reconstruction

Check reconstruction output:

```bash
python -m src.cli.main validate /path/to/dataset --stage reconstruction
```

#### 6. Validate LAS/LAZ

Check final point cloud:

```bash
python -m src.cli.main validate /path/to/dataset --stage las
```

#### 7. Run All Validations

```bash
python -m src.cli.main validate /path/to/dataset --stage all
```

### Help

```bash
python -m src.cli.main --help
```

## Configuration

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `LOG_LEVEL` | Logging level | `INFO` |
| `ODM_DOCKER_IMAGE` | ODM Docker image | `opendronemap/odm` |

Set via:
```bash
export LOG_LEVEL=DEBUG
export ODM_DOCKER_IMAGE=opendronemap/odm:v2.8.1
```

### Validation Thresholds

- **Minimum images**: 5 (configurable in `PreflightValidator`)
- **Minimum points**: 10,000 (configurable in `ReconstructionValidator`)

## Output Files

After successful pipeline run:

| File | Description |
|------|-------------|
| `outputs/pointcloud.las` | Normalized LAS (EPSG:4326, LAS 1.4, Point Format 7) |
| `outputs/pointcloud.laz` | Compressed LAZ |
| `outputs/reconstruction_report.json` | ODM execution report |
| `outputs/metadata.json` | Reproducibility metadata |

## Troubleshooting

### "Module not found" Error

Set PYTHONPATH:
```bash
export PYTHONPATH=/path/to/project
```

### Docker Not Available

Some environments don't support Docker (containers, certain VMs, CI/CD):
- Use Tier 1 (Python only) for validation and geo.txt generation
- Run reconstruction on a separate machine with Docker
- Or use a cloud-based Docker service

### Docker Permission Denied

Add user to docker group:
```bash
sudo usermod -aG docker $USER
# Then logout and login back
```

### PDAL Not Found

Install PDAL:
```bash
# Ubuntu/Debian
sudo apt-get install pdal

# macOS
brew install pdal
```
